# 초등학교 특별실·전담수업 시간표 시스템 설계

작성일: 2026-05-14

---

## 1. 프로젝트 개요

초등학교의 특별실 사용 시간표와 전담수업 시간표를 통합 관리하는 웹 기반 시스템.  
시간표 담당자 1~2인이 로컬에서 운영하는 단일 학교용 관리 도구로 시작하며, 이후 서버 배포를 통한 멀티유저 전환이 가능한 구조로 설계한다.

---

## 2. 확정된 요구사항

| 항목 | 결정 |
|------|------|
| 범위 | 특별실 + 전담수업 통합 시스템 |
| 실행 환경 | 로컬 웹 앱 (브라우저) |
| 데이터 저장 | PostgreSQL (로컬 설치) |
| 자동 배정 | 반자동 — 충돌 감지 + 수동 드래그앤드롭 편집 |
| 반복 배정 | 캘린더 앱 방식 (주기·요일·종료 조건 설정) |
| 학사일정 | 사용 일수 집계 + 방학 중 배정 차단 + 예외 지정 |
| 단발 예약 | 정기 시간표 외 단발성 예약 지원 |
| 시간 단위 | 교시 기본 + 학교별 시작·종료 시각 설정 |

```
src/
  app/
    (setup)/          # 기준정보 관리 페이지

    (calendar)/       # 달력형 보기 페이지
    (academic)/       # 학사일정 관리 페이지
    api/              # 파일 처리 전용 (엑셀 내보내기 등)
  features/
    rooms/            # 특별실 관리
    classes/          # 학년·학급 관리

    schedule/         # 배정 편집·조회
    reservations/     # 단발성 예약
    academic-calendar/# 학사일정

prisma/
  schema.prisma
```

## 5. 데이터 모델

### 주요 엔터티

```
SchoolTerm (학년도·학기)

Grade (학년)
  └── ClassGroup (학급)
        └── homeroom_teacher → Teacher

Subject (과목)
  ├── 이름, 종류(전담/일반), 특별실 필요 여부
  └── 연결 특별실 목록 (복수 가능)

Teacher (교사)
  ├── 이름, 종류(담임/전담/겸임)
  ├── TeacherSubject (교사-과목 다대다)
  │     └── 한 교사가 여러 과목 담당 가능
  └── TeacherUnavailability (비가용 시간)

SpecialRoom (특별실)
  ├── 이름, 종류, 동시 사용 가능 학급 수(capacity), 비고
  └── RoomUnavailability (비가용 시간)

Period (교시)
  └── 교시 번호, 시작 시각, 종료 시각

AcademicEvent (학사일정)
  └── 종류(시업식·여름방학식·겨울방학식·종업식 등), 날짜, 예외 허용 여부

ScheduleRule (정기 배정 규칙)
  ├── 특별실, 학급, 과목(선택), 담당교사(선택), 교시
  ├── 반복 주기: 숫자 + 단위(일/주/개월)
  ├── 반복 요일: 월·화·수·목·금 다중 선택
  └── 종료 조건: 없음 / 날짜 / N회 반복

ScheduleEntry (실제 배정 인스턴스)
  ├── 날짜, 교시, 특별실, 학급, 과목(선택), 담당교사(선택)
  ├── 출처: 정기규칙 ID / 단발예약 / 수동
  └── 상태: 정상 / 예외취소 / 강제배정

Reservation (단발성 예약)
  └── 날짜, 교시, 특별실, 학급, 과목(선택), 담당교사(선택), 사유
```

### 핵심 설계 결정

- `ScheduleRule`은 반복 패턴만 저장, 실제 달력 인스턴스는 `ScheduleEntry`로 전개
- 방학 중 배정 차단은 엔진이 `AcademicEvent`를 참조해 자동 처리, 예외 허용 시 `ScheduleEntry.상태 = 예외허용`
- `SpecialRoom.capacity`로 과학실 분반 등 동시 사용 처리

## 6. 화면 구조

### ① 설정 (기준정보 관리)
- 특별실 목록 직접 등록·수정·삭제 (과학실, 컴퓨터실, 음악실, 미술실, 시청각실, 체육관, 운동장, 도서실 등)
- 학년·학급 등록, 학급 수 설정
- 교사 등록 (담임/전담/겸임 구분, 담당 과목 다중 연결)
- 과목 등록 (전담/일반 구분, 필요 특별실 연결)
- 교시 설정 (교시 번호, 시작·종료 시각)
- 학사일정 등록 (시업식, 여름방학식, 겨울방학식, 종업식 등)

### ② 시간표 편집기 (메인)
- 주간 그리드: 가로=요일(월~금), 세로=교시
- 좌측 패널: 특별실 필터, 미배정 학급 목록
- 그리드 셀: 학급명 + 과목 + 교사 표시
- 드래그앤드롭으로 배정 이동
- 충돌 즉시 표시 (빨간 테두리 + 사유)
- 정기 배정 규칙 생성 시 반복 설정 다이얼로그

### ③ 달력형 보기
- 월별 달력에 특별실별 배정 현황 표시
- 학사일정 이벤트 오버레이 (방학·행사일)
- 날짜 셀 클릭 시 상세 배정 목록
- 단발성 예약 추가

### ④ 목록형 보기
- 특별실별 / 학급별 / 교사별 필터
- 날짜 범위 선택
- 특별실 사용 일수·횟수 집계 (학사일정 기준 실수업일)
- 엑셀 내보내기

### ⑤ 단발성 예약
- 날짜·교시·특별실·학급·과목·교사·사유 입력
- 충돌 즉시 검사
- 달력형 보기와 통합 표시

## 7. 배정 엔진

### 7.1 반복 규칙 전개 (Rule Expander)

- `ScheduleRule` → 학기 전체 `ScheduleEntry` 생성
- `AcademicEvent` 참조하여 방학·휴일 자동 제외
- 예외 허용 날짜는 통과
- 전개 범위: 학기 전체 또는 사용자 지정 기간

### 7.2 충돌 검사 (Conflict Checker)

하드 제약 검사:
- 같은 날짜·교시에 동일 특별실 중복 (capacity 초과 포함)
- 같은 날짜·교시에 동일 학급 중복
- 같은 날짜·교시에 동일 교사 중복
- 특별실·교사 비가용 시간 위반

동작 방식:
- 드래그앤드롭 이동 시 즉시 검사 → UI 결과 반환
- 강제 배정 시 경고만 표시하고 저장 허용

### 7.3 처리 흐름

```
규칙 저장
  → Rule Expander (학기 전체 전개)
  → Conflict Checker (일괄 검사)
  → 결과 저장 + 충돌 목록 반환

드래그앤드롭 이동
  → Conflict Checker (해당 슬롯만 즉시 검사)
  → 충돌 없음: 저장 / 충돌 있음: UI 경고
```

## 8. MVP 범위

- 기준정보 관리 (특별실, 학년·학급, 교사, 과목, 교시, 학사일정)
